<static> GNN.Base.covers(lhs, rhs)
Checks if the first values covers the second value, in other words, the first value is observably equivalent to the second value, if one considers only the structure of the second value. It checks equality over values of enumerable properties. Input values may have cyclic references. 
var x = { a: 1, b: 2, c: 3 };
GNN.Base.covers(x, { a: 1, b: 2 }); // => true
GNN.Base.covers(x, { a: 1, b: 2, c: 3 }); // => true
GNN.Base.covers(x, { a: 1, b: 3 }); // => false
GNN.Base.covers(x, { a: 1, b: 2, c: 3, d: 4 }); // => false

var y = { a: 1, b: 2, c: 3, d: 4 };
y.a = y;
var z = { a: 1, b: 2 };
z.a = z;

GNN.Base.covers(y, z); // => true

<static> {object} GNN.Base.fmerge(fun, objs)
Merges the objects by the given function.

For input obj1, obj2, ...rest, it merges obj1 and obj2 into obj1 and computes fmerge(fun, obj1, ...rest) iteratively. In each iteration, it calls fun(obj1[p], obj2[p], p) for each enumerable property p in obj2 and updates the property p of obj1 by the return value of fun if it is not undefined.

If the first element of objs is null, it merges rest of objects into a new object {}. If the other elements of objs are null, they are just skipped.

GNN.Base.fmerge(function(x,y,k) {
    return x || y;
}, null, {a:1,b:2}, {c:3}); // => {a:1,b:2,c:3}
GNN.Base.fmerge(function(x,y,k) {
    return x || y;
}, {a:1,b:2}, {b:3}); // => {a:1,b:2}
Parameters:
{function} fun
A function takes two values and returns a value or undefined.
{...object} objs
Zero or more objects to be merged.
Returns:
{object} The value whose reference is identical to the first element of objs.

<static> {boolean} GNN.Base.sim(lhs, rhs, axiom, transition)
Checks if there is a simulation over the given labelled state transition system.

The labelled state transition is given by values as states and indexes of a return value of transition as labels. The return value of transition is an array of pairs of next states.

The overall result is true if and only if axiom(lhs, rhs) returns true and sim(x[i], y[i], axiom, transition) returns true for every i where transition(lhs, rhs) returns [ [ x[0], y[0] ], ..., [ x[i], y[i] ], ... ].

For any states x1 and y1, sim(x1, y1, axiom, transition) = true implies sim(x2, y2, axiom, transition) = true for every pair of states x2, y2 returned by transition(x1, y1). This means that there is a relation called simulation over the transition system and the relation is actually a bisimulation if axiom is symmetric.

This method is useful for checking observable equivalence.

Parameters:
{*} lhs
{*} rhs
{function} axiom
A function which takes two states and returns a boolean value.
{function} transition
A function which takes two states and returns an array of pairs of states.
Returns:
{boolean}
